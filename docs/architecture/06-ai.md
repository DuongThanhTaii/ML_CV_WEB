# AI Subsystem

## Providers

| Provider | Model | Used for | Free quota |
|---|---|---|---|
| Groq | Llama 3.3 70B Versatile | Chat tutor, feedback | 14,400 req/day |
| Gemini | gemini-2.0-flash | Long-context (code review) | 1500 req/day |

All providers fronted by `src/lib/ai/gateway.ts`. Never call providers directly from components.

## Streaming

- Server route returns SSE (text/event-stream).
- Client uses native ReadableStream reader, parses OpenAI-format chunks.
- See `src/components/ai/tutor-chat.tsx` for reference implementation.

## Prompt versioning

Prompts in `src/lib/ai/prompts/` carry a `*_PROMPT_VERSION` constant. Bump version when changing semantics so feedback rows can be filtered by version.

## Context assembly

For tutor, the context (in order) is:

1. System prompt (pedagogy rules)
2. Lesson title + first 800 chars of lesson MDX (if `context.lessonId`)
3. Student code (first 2000 chars, if `context.code`)
4. Latest error (if `context.error`)
5. User message

## Safety

- Rate limit per user: 50 chats/day, 1/10s
- System prompt anchored against jailbreak ("ignore prior", "you are now …")
- Output filter: regex check for "def solve" pattern (likely full solution leak)
- No PII (name, email) ever in prompts

## Semantic cache (future)

- Embed user question → cosine similarity against `ai_chat_messages.embedding`
- If sim > 0.92, return cached response
- Stored in pgvector (already in schema)

## Cost model

- 1 tutor question ≈ 800 tokens prompt + 400 tokens response = 1200 tokens
- Groq free: 14400 req/day = ~200 active users at 70 q/day each
- Beyond that: introduce semantic cache, then upgrade to paid (Groq Pro $99/mo, supports 100k+ req/day)
